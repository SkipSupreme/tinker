# Tinker v1: Master Plan

> **For future Claude:** this is a strategy doc, not an implementation plan. Per-phase TDD execution plans (using @superpowers:writing-plans) land as separate docs when each phase starts, e.g. `2026-05-??-phase-0-thin-slice.md`.

**Goal:** Ship a Math-for-ML learning app for adult technical learners. Web-first, wrapped in Capacitor for iOS. Built on the widget engine we already have.

**Architecture:** Astro 6 + Svelte 5 + MDX lessons on Cloudflare (Workers + D1 + R2 + Durable Objects). svelte-mafs widgets embedded inline. ts-fsrs at concept granularity with FIRe-style prerequisite credit. Claude Sonnet 4.6 via AI Gateway for Socratic tutoring, Opus 4.6 + Pyodide/SymPy for verified problem generation. Capacitor 8 wrap for App Store after web monetization is validated.

**Precursor:** widget engine shipped. See [`2026-04-23-svelte-mafs.md`](./2026-04-23-svelte-mafs.md) for the widget-engine plan we just executed.

## 1. Frozen decisions

Decisions in this section are *settled*. Change them only on evidence, not on vibes.

### 1.1 Wedge: Math for ML
Linear algebra → vector calculus → probability → statistics → gradient descent → neural-net fundamentals.

**Why.** Adult technical audience (highest CAC-to-LTV in consumer ed-tech). Natural overlap with our skill set. Verifiable answers (symbolic + unit-testable). 3Blue1Brown-ecosystem-warm audience. X/Twitter math community already engaged. **White space:** Brilliant caps at intro, Math Academy is rigorous-but-dry + web-only, Khan is free-and-sprawling, nobody serves the adults-upskilling-for-ML segment on mobile with a widget-first pedagogy.

**Not v1:** CS / algorithms (Phase 7+), probability standalone, physics, chemistry, humanities.

### 1.2 Price: $129/yr + $399 lifetime
- **Annual $129/yr** (7-day free trial). $10.75/mo effective. Below Brilliant ($162), below Duolingo Super ($84) on value density. Anchor: "cheaper than Brilliant, deeper than Duolingo, less dry than Math Academy."
- **Lifetime $399.** Breaks even at 3.1 years of annual. Solo-dev cash-flow bump at launch + high-intent buyer signal. Setapp / Tana / Obsidian Publish validated the pattern for prosumer subscription tools.
- **Family $199/yr for 6 seats** via Apple Family Sharing. Research-backed churn reduction (30–40% lower than solo subs).

**No freemium with content throttle** like Brilliant's "2 lessons/day", reviewers correctly call that a dark pattern on adult audiences. **Trial model** instead: 7-day full access, convert-or-lock. RevenueCat 2025 data: trial-to-paid EdTech 22–25%; D35 retention 12.11% trial vs. 2.18% freemium.

### 1.3 Stack
- **Web:** Astro 6 + Svelte 5 + MDX (content collections with Zod schemas). KaTeX at build time via `remark-math` + `rehype-katex`. svelte-mafs widgets via `<Component client:visible />` islands.
- **Edge:** Cloudflare Workers (Hono for REST, native Workers WebSocket). D1 (shared content + metadata). R2 (assets + Pyodide packages). Durable Objects (per-user review log, SQLite-backed).
- **AI:** Anthropic Claude (Sonnet 4.6 tutor; Haiku 4.5 classifier; Opus 4.6 for batch problem gen) via Cloudflare AI Gateway. Vectorize for lesson RAG. Pyodide 0.27 + SymPy in a Web Worker, browser-side, for symbolic verification.
- **Native:** Capacitor 8. RevenueCat for IAP.
- **SRS:** `ts-fsrs` (FSRS-6), concept-granularity cards.
- **Auth:** WorkOS AuthKit (email magic-link + Google + Apple). JWT at the edge.
- **Analytics:** own-event-log (D1 writes mirrored to R2 Parquet). Self-hosted Umami for anonymous marketing analytics. PostHog self-hosted on Fly after 5k MAU. No third-party cookies.
- **Newsletter:** Beehiiv (the waitlist emails go here).

### 1.4 SRS + knowledge graph
- **FSRS cards = concepts**, not individual problems. Any problem attempt tagged to a concept is a review of that concept.
- **Rating mapping:** correct + no hints + <50% expected time = Easy · correct + no hints = Good · correct + ≥1 hint = Hard · incorrect = Again.
- **FIRe-style prerequisite credit** (Math Academy's trick): getting a hard problem right fractionally credits prerequisite concepts on the DAG. Implemented as graph traversal; ~300 LOC. This is the pedagogical differentiator over Brilliant.
- **Knowledge graph:** hand-built 120-concept DAG for the Math-for-ML wedge. YAML source of truth. LLM proposes edges, I hand-verify every one. This is the moat; do it serially and carefully.
- **Target retention:** 0.88. User-visible knob.
- **Mastery gate:** ≥0.75 concept retention on prerequisites before a lesson unlocks.

### 1.5 AI tutor
- **Explicit opt-in** before any user input hits Claude. App Review Guideline 5.1.2(i) is non-negotiable, consent flow must be standalone, not bundled with other permissions, with clear opt-out.
- **Structured tool-use for hints**, never free-text parsing: `{ hintLevel: 1 | 2 | 3, text, revealsAnswer: boolean }`.
- **Socratic system prompt** (inspired by Claude Learning Mode). Haiku post-classifier redacts answer-leakers.
- **Prompt caching** on lesson context (90% savings on repeat turns).
- **Never grade authoritatively.** SymPy / exact-match / unit tests decide pass-fail. AI provides feedback, not scores.
- **Daily per-user spend cap**, enforced at the AI Gateway.

### 1.6 Content license discipline
- 100% original, OR **OpenStax CC BY** with per-book license check (many OpenStax books are CC BY-NC-SA → unusable for a paid app). Safe subset: verify each before embedding.
- **Never embed** Khan Academy (CC BY-NC-SA), MIT OCW (CC BY-NC-SA), CK-12 (CC BY-NC), Wikipedia (CC BY-SA copyleft).
- Images: original, US-government public-domain, NASA, or CC BY with visible attribution.
- Footer "Credits" page lists every third-party source, license, and link.

### 1.7 Pedagogy commitments
- **Interleaving:** cumulative review mixes concepts across modules. Within-lesson practice may block; cross-module review never does.
- **Worked examples with fading:** for novices, first occurrence is a fully-worked example; second is a completion problem; third is cold practice. Expertise-reversal effect kicks in fast, don't over-scaffold.
- **Flow targeting:** adaptive difficulty aims at 80% correctness. Let the user drag the dial ±5%.
- **No streaks for 30 days** after signup. After 30 days, opt-in streaks with free daily freezes. Sundays are always freeze days. No streak-anxiety notifications.
- **WCAG 2.2 AA** from the start. Retrofitting accessibility after the fact is multiples of the cost of committing up-front.

### 1.8 Mobile strategy
- **Capacitor 8 wrap.** Not React Native (discards our Svelte codebase), not native Swift (rebuilds the widget engine), not Tauri Mobile (not first-class per Tauri team), not PWA-only (iOS push opt-in ~16% vs. 40–70% native; no App Store discovery).
- **Ship native features** to clear App Review Guideline 4.2: native tab bar, APNs push, offline fallback (critical, a white-screen-on-network-loss is auto-reject), haptics, share sheet, deep links.
- **Web first.** Capacitor only after web revenue is validated (target: ≥100 paid subs). Premature mobile is a common solo-dev trap.

### 1.9 What we will NOT do in v1
Each is sized and ruled out per research:
- Community forum (Brilliant tried, killed it, moderation-heavy, pedagogy-diluting)
- Live cohorts / Maven-style instruction (margin-compressed commodity)
- User-generated content / marketplace (same)
- Peer review (pointless for deterministic-verification math)
- Voice-first AI for kids (speech recognition on child voices still unreliable per Synthesis Tutor Dec 2025 reviews)
- Camera / AR scan-and-solve (Photomath owns it, it's anti-learning anyway)
- Vision Pro (consumer install base negligible through 2026; "Designed for iPad" gives us the optionality free)
- Kids category on App Store (third-party analytics ban + permanent obligations; stay in Education)
- Multi-subject at launch (subject sprawl = four bad products)
- SEO-heavy content strategy (zero-click landscape; 46.7% CTR reduction from AI Overviews)
- Paid acquisition before LTV proof (industry EdTech CAC $806–$1,617; no paid channel until LTV clears that bar organically)
- VC (market is cold: $20.8B 2021 → $1B 2024; wrong incentives for the business we're building)
- Open-sourcing the lessons (engine can be open later; lessons are the moat)
- Hiring before 5k MAU with proven retention

## 2. Build phases

Write ONE lesson end-to-end. Ship to `/lessons/what-is-a-derivative`. Use it myself daily. Does the MDX-plus-Svelte-widgets format actually teach? Does the widget engine cover what I reach for?

**Deliverables:**

- `apps/docs/src/components/Lesson.astro`: step-gated reader with progress bar

**Exit criteria:**
- I've done the lesson 3 times without cringing at format friction
- ≥2 of the 3 widgets use svelte-mafs primitives unchanged (not custom one-offs)
- Total lesson render ≤100 KB gzipped JS over the wire (first load), ≤30 KB per subsequent lesson

**Why start here:** the research report's prescription is *"write the first lesson end-to-end in MDX + Svelte + KaTeX as a spike. That spike will tell you more than the next ten reports."* Agreed. This is that spike.

### Phase 1: Content infrastructure
Scaffold the rest of the content will ride on. 3 more lessons to pressure-test.

**Deliverables:**
- Content-collection types: `title`, `module`, `estimated_minutes`, `prereqs[]`, `concepts_covered[]`, `license`

- 3 more lessons: *"Vectors as lists of numbers"*, *"Matrix-vector multiplication is a linear combination"*, *"Derivative of a function: limits, intuitively"*

**Exit criteria:**
- 4 published lessons. Zero one-off components per lesson (widgets come from svelte-mafs or the step-gate library, nothing lesson-specific)
- Lesson authoring workflow feels repeatable.

### Phase 2: User accounts + progress persistence
Replace localStorage with accounts + sync.

**Deliverables:**
- WorkOS AuthKit: magic-link email, Google, Apple. JWT on the edge.
- D1 schema: `users`, `lessons`, `lesson_events` (append-only), `problem_events` (append-only), `concepts`, `prereqs`
- Durable Object per user: owns the user's event log, SQLite-backed. Recomputes derived state on write.
- Sync protocol: WebSocket, lamport-clocked events upstream, authoritative state snapshots downstream. ~300 LOC per the research report's architecture.
- Client store: Dexie on IndexedDB (v1). Upgrade to SQLite-WASM + OPFS (Safari 17+) in Phase 6.
- Account management: sign in, sign out, delete account (full GDPR wipe).

**Exit criteria:**
- A user can sign in on desktop, complete a step, resume on mobile web with progress preserved in <2s
- Account-deletion wipes the Durable Object, the D1 user row, and all event rows
- Offline session: disconnect wifi, complete 3 steps, reconnect, events sync to DO without loss or duplication

### Phase 3: FSRS + knowledge graph
The pedagogical core.

**Deliverables:**
- `docs/knowledge-graph/math-for-ml.yaml`: hand-built 120-concept DAG. Each concept: id, name, module, prereqs, encompassings, lesson(s) covering it.
- LLM tool that proposes edges given (concept, existing graph). Hand-verify every edge (no automated ingestion, this IS the moat).
- `ts-fsrs` integrated server-side in the DO. One card per (user, concept).
- Rating mapper (see decision 1.4).
- FIRe prerequisite credit: ~300 LOC graph traversal, fractional stability update on prereqs.
- Practice queue: due reviews by priority, interleaved across modules.
- Mastery gates: lesson unlocks only when prereq retention ≥0.75.

**Exit criteria:**
- A user with enough reviews logged to stabilize FSRS sees sensibly-spaced reviews (manually inspect 20 cases against FSRS default expectations)
- Prerequisite trickle-credit: solve a hard MLE problem, confirm `linear-algebra/expectation` and `probability/random-variable` gain partial stability
- Zero reviews scheduled outside the [user-visible] `min_interval_hours`: `max_interval_days` range

### Phase 4: AI tutor
Opt-in, Socratic, safe.

**Deliverables:**
- Consent flow per Guideline 5.1.2(i): standalone screen, clear language, opt-in before first Claude transmission, persistent opt-out, documented in `/privacy`
- `apps/docs/src/lib/tutor.ts`: client shim calling `/api/tutor/hint`
- Worker route `POST /api/tutor/hint` via AI Gateway (caching, rate limits, logging)
- Sonnet 4.6 with prompt caching on lesson context
- Structured tool-use: `{ hintLevel, text, revealsAnswer }`: never free text
- Haiku 4.5 post-classifier: redacts answer-leakers before return
- Per-user daily spend cap at the Gateway
- `/hints` settings page: history, opt-out, spend-to-date

**Exit criteria:**
- I can complete a fresh ML math problem using only hints, without the tutor ever revealing the answer
- Gateway logs show 0 transmissions for users who haven't opted in
- Redaction classifier catches ≥95% of synthetic "here's the answer: X" injections in a test set

### Phase 5: Problem generation + correctness evals
Infinite verified practice.

**Deliverables:**
- `scripts/generate-problems.ts`: Opus 4.6 via Batch API (50% off) generates candidates per concept
- `apps/docs/src/lib/verify.worker.ts`: Pyodide + SymPy in a Web Worker. Independently solves each candidate. Reject if disagree.
- Accepted problems stored in D1 with (statement, answer, solver-steps, concept-ids[], difficulty, generated-at, verified-by-version)
- Nightly batch: 500 candidates → ~200 accepted (expected drop rate)
- Golden-set: 500 hand-authored canonical problems per concept, Promptfoo evals nightly
- Regression alert: page if pass-rate on golden set drops >3% week-over-week

**Exit criteria:**
- ≥1,000 verified problems in D1 covering all 120 concepts
- Zero escaping-wrong-answer incidents in production use
- Golden-set pass rate baseline established, alert plumbing tested

## 3. Content plan: Math for ML v1

60 lessons across 6 modules. Each lesson: 8–15 interactive steps, 500–1500 words, 3–5 worked examples, 5–10 practice problems × 3–5 variations each, ~30 min expected time.

### Module 1: Linear algebra (12)
Vectors as lists of numbers · Vector addition + scalar multiplication · Dot product = angle · Basis intuition · Matrix = linear map · Matrix-vector multiplication (linear-combination interpretation) · Matrix-matrix multiplication · Identity + inverse · Determinant = volume · Rank + null space · Projections · Least squares

### Module 2: Vector calculus (10)
Functions of multiple variables · Partial derivatives · Gradient = steepest ascent · Directional derivatives · Jacobian · Hessian (2D intuition) · Chain rule: single-variable composition · Chain rule: multivariable · Matrix-calculus identity cheat-sheet · Common ML gradients (log-sum-exp, softmax)

### Module 3: Probability (10)
Sample spaces · Random variables · Expectation · Variance · Independence · Conditional probability · Bayes' rule · Bernoulli → Binomial · Gaussian (1D + nD) · Joint distributions

### Module 4: Statistics (10)
Sample mean · Central limit theorem · Estimators + bias + consistency · Maximum likelihood · KL divergence · Cross-entropy · Regularization (L1/L2) · Bias-variance tradeoff · Linear regression derivation · Classification metrics (precision/recall/F1/ROC)

### Module 5: Gradient descent + optimization (8)
Loss landscapes · Gradient step · Learning rate (visual) · Stochastic vs. batch vs. mini-batch · Momentum · Adam · Convergence criteria · Non-convex pitfalls

### Module 6: Neural-net fundamentals (10)
Neuron = weighted sum + activation · Layers · Forward pass · Backward pass · Backprop derivation (small worked example) · ReLU vs. sigmoid · Softmax · Single-hidden-layer network · Universal approximation (intuitive) · Why deep > wide

**Content pipeline:** see `docs/plans/2026-04-24-research-prompts.md`. Deep-research briefs per module → Claude-drafted MDX lessons → hand-edit for voice → SME contractor reviews each module before publish. No more "solo authoring every word", that's a pre-LLM assumption.

### Authoring workflow (per lesson)
1. Concept must already exist in DAG with verified prereqs.
2. Outline the 10 "moments of understanding" this lesson produces.
3. Write step-gated MDX with widget embeds inline.
4. Hand-author 5 canonical problems (these become part of the golden set).
5. LLM generates 50 variations; SymPy verifies; pick top 30 by difficulty spread.
6. Dogfood: can I do the lesson tired? If not, rewrite the confusing step.
7. SME review before publish.

## 4. Hard problems we haven't solved

These aren't show-stoppers, but they're the places we'll learn the most.

1. **FSRS → interactive-problem rating mapping** is judgment-heavy. The heuristic in §1.4 is an initial guess. Track per-user retention calibration; if a user's "correct without hints" retention diverges from target by >10%, tune the mapping. Revisit at 500 users.
2. **Knowledge-graph labor.** 120 concepts × LLM-propose + hand-verify every edge. Math Academy hand-built a 2500-node graph as their primary differentiator, this work is the moat, worth doing serially. If the scope proves too wide, cut to 80 concepts.
3. **AI correctness evals at small scale.** Golden-set approach is stable but labor-heavy to bootstrap. Use Promptfoo; start with 50 golden problems per concept (not 500) for v1; widen over time. Brilliant has an open job posting for this, confirming even they haven't nailed it.
4. **App Store 5.1.2(i) consent UX.** Clean opt-in + opt-out flow must read unambiguously to reviewers. Dogfood in Phase 4; expect 1–2 rejection cycles in Phase 6.
5. **Offline course download with Pyodide.** 11MB core + 2–5MB per package. Precache after first lesson that loads it; user-facing "download for offline" button per module; LRU eviction when storage pressure hits.
6. **Subscription reconciliation across iOS + web.** RevenueCat solves iOS. Web is Stripe Checkout. Tie via email. Expose a "Manage subscription" link that picks the right path per platform.

## 5. Risk register

- **Content treadmill.** 60 lessons is more content than any solo human author can ship in parallel with building infrastructure. *Mitigation:* deep-research prompts + LLM drafting + SME review (see `2026-04-24-research-prompts.md`); thin-slice MVP at 15 lessons; open paid waitlist at MVP, not v1.
- **Retention graveyard.** Education day-30 retention ~2% median (Netflix is 98.2% *monthly*). *Mitigation:* FSRS + mastery gates + no-streak-first-30-days + daily-review email (not push) for the adult audience.
- **CAC > LTV in early stage.** EdTech CAC $806–$1,617 is industry context. *Mitigation:* no paid acquisition until LTV clears that bar organically. Organic channels (newsletter + X + HN/PH). Re-evaluate at 2k MAU.
- **Scope creep.** Commit hard to Math for ML. Adjacent subjects are *future products*.
- **WebView rejection.** Apple 4.2 is strictly enforced. *Mitigation:* build native-feature list in Phase 6 day 1; do not wing it at submission time.
- **COPPA via state age-verification laws.** TX (Jan 2026) + UT + LA + CA (Jan 2027) pass age signals to apps, silent COPPA trigger. *Mitigation:* default users to adult; silo age signals; if declared <13, route to verifiable-parental-consent flow (even though our target is adults).
- **WCAG 2.2 AA retrofit cost.** Multiples of the commit-now cost. *Mitigation:* every widget a11y-tested at commit, every lesson keyboard-navigable before publish.
- **LLM billing runaway.** *Mitigation:* per-user daily spend cap enforced at the Gateway; fail-closed.
- **Single LLM vendor dependency.** *Mitigation:* abstract tutor behind an interface. Claude is primary; OpenAI swappable backend in Phase 4.
- **Trademark collision on "Tinker".** Immediate USPTO + CIPO search (see §7).
- **Content-licensing poison.** *Mitigation:* the §1.6 discipline. If in doubt, don't embed.
